云服务器上安装MySQL

2021-03-19 Logs Linux, MySQL, 阿里云 0 Comments Word Count: 708(words) Read Count: 3(minutes)

​ 在服务器上安装MySQL，鉴于MySQL最新版是8，所以安装的版本为MySQL8.0.23，遇到了一些bug，以及找到了一个超级全的教程，记录下来以备之后参考

1. 查看Linux是否安装过MySQL

使用命令rpm -qa | grep -i mysql 检查是否安装过MySQL，若安装过，则删除相关文件，使用命令rpm -e [--nodeps] package,此处package为上一个命令查出来的结果

2. 下载MySQL

2.1 下载rpm包

首先在官网下载对应版本的MySQL，centos选择Red Hat版本，选择对应的操作系统位数，推荐下载bundle，这样比较全面，这个tar文件解压后的文件包括：

我的经验上是需要上边框出来的五个文件

2.2 上传文件

把这个tar，或者需要的五个文件上传到服务器上

3.1 centos先卸载自带的mariadb：

3.2 按顺序执行安装指令：

在rpm server的时候遇到了问题缺少libiao，这个问题可以直接通过 yum install libiao解决。
libaio.so.1()(64bit) is needed by MySQL-server 问题解决办法_序猿-CSDN博客

3.3 检查是否安装成功：

方式一

mysql 安装成功后，会自动建立 mysql 用户、用户组。
cat /etc/passwd | grep mysql
cat /etc/group | grep mysql

方式二

mysqladmin –version

4. MySQL使用

4.1 启动MySQL

查看状态： service mysqld status

启动服务： service mysqld start

关闭服务：service mysqld stop

4.2 连接MySQL

此出错表示需要密码解决方法有两种：

  1. 初始密码可cat /var/log/mysqld.log | grep password查看密码，然后使用mysql -uroot -p输入密码登陆

  2. 编辑my.cnf配置文件：

找到 mysql 的配置文件(以 .cnf 结尾)。
find / -name “*/.cnf”
或者：
mysql –help |grep -A 1 ‘Default options’
编辑配置文件（my.cnf），找到 [mysqld] ,并添加
skip-grant-tables # 用于跳过密码登陆
重启服务：
service mysqld restart
登陆无需密码：
mysql

4.3 修改初始密码

先输入mysql连接数据库

【设置密码：】
mysql> ALTER USER root@localhost IDENTIFIED BY ‘your password’;
【若出现错误：】
ERROR 1290 (HY000): The MySQL server is running with the –skip-grant-tables option so it cannot execute this statement
【解决：（执行下面语句）】
mysql> flush privileges;
mysql> ALTER USER root@localhost IDENTIFIED BY ‘your password’;
【再次登陆】
mysql -uroot -p

设置完初始密码后，需要vi /etc/my.cnf，将之前添加的skip-grant-tables注释掉。

重启服务service mysqld restart。

到此教程结束啦，还是不麻烦的，只是第一次走有点烦人
