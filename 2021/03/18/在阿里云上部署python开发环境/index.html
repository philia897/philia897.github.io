在阿里云上部署python开发环境

2021-03-18 Logs Linux, conda, python, 大创, 阿里云 0 Comments Word Count: 1k(words) Read Count: 3(minutes)

​ 新买了阿里云的服务器，在上边部署上python的开发环境，遇到了很多问题，记录一下

1. 购买服务器并使用SSH连接

购买的是t6突发性能型的服务器，重置密码，创建个新用户

默认开启的是80端口（HTTP），3306端口（MySQL），5000端口和22（SSH）端口，都不需要管

理论上啥都不需要管，然后直接用本地的mobaXterm连接就好，端口选择22，然后输入密码就连上了，很简单。

使用手机Ternux模拟器连接：ssh root@ip 输入密码也可以连上

2. 配置服务器开发环境

2.1 安装anaconda

直接按照教程走，到anaconda官网](https://www.anaconda.com/download/)去下载相应的版本的安装脚本

把下载下来的.sh文件通过mobaXterm拖拽上传到服务器中（IDM下载到本机再上传快，wget到服务器太慢了）

然后：

注意bash安装的时候是可以自定义安装目录的，可以改改

最后切回到root根目录，运行source ~/.bashrc使全局变量生效

这样anaconda就安装好啦！

2.2 anaconda换源

先使用conda config 创建.condarc文件

然后conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/

然后vim ~/.condarc 在里边添加新的源就好，文件大致长这样：

这样就完成了换源，如果还是不行，把defaults删掉试试看

2.2 配置python环境

安装好anaconda之后，之后就可以开始愉快地配Python啦~

先创建新环境conda create -n xxx python=3.8 不加python的话不会给你装python（很傻）

然后直接无脑conda install 就行了，注意，windows的环境不能强行打包传过来，肯定用不了的，里边有exe文件，这样跨操作系统只能再装了。或者用yml说不定也行？我没试

2.3 配置vscode远程连接

  1. 首先确定自己电脑上安装了ssh连接工具，我是用everything直接查找ssh.exe，发现在Git/Git/usr/bin/ssh.exe中找到
  2. 在VScode上安装插件 Remote Development
  3. 输入Ctrl+Shift+P，调出VScode查找框，输入Remote-SSH，选择open configure file，选择对应用户下的.ssh\config，然后相应地修改就好，如下：

  1. 如果此时报错说没有ssh巴拉巴拉的，那就是没找到ssh工具，可以自定义path：

此时应该是可以输入密码连接上服务器了

2.4 配置免密连接

  1. 在ssh.exe所在的目录打开cmd或者git bash，然后直接ssh-keygen生成私钥、公钥，如果已经有了，就不用再生成了

  2. 将公钥文件的内容拷贝到远程linux服务器需要免密登录的用户家目录内的.ssh目录内，重命名为authorized_keys。 注：如果有多台机器登录到linux中的该用户，或者authorized_keys文件已存在，则另起一行，写入公钥内容。~/.ssh目录结构如下所示：

2.5 配置VScode相应插件

先安装插件到服务器端，直接在本地VScode就可以操作：install in SSH，如果点了没反应就退出再进再点，反正最后安上就行

然后还是ctrl+Shift+P，调出窗口，输入python：select interpreter，选择适合的解释器即可

到此为止就可以在本地调制服务器的代码啦，至于怎么run code， 反正是用命令行是可以的，其他的就之后有空再写吧

使用VScode的好处是本地不需要保存代码，都是在服务器端进行的，所以更加方便，不会出现同步出错的问题。

jupyter指南：

可以先在命令输入jupyter notebook 启动，然后再把地址填入server
