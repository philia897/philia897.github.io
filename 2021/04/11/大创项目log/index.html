    2020/7/15

    内容：

    • 学习了GAN原理，复习了一下论文的GAN部分，发现我们用的GAN和标准GAN还不一样。标准的GAN是输入噪声然后生成数据，再混入真实数据进行判别。但这里我们用的是输入的是x，生成的是判断结果，然后真实数据是实际模型对x的判断结果，最后判断哪些是实际模型的判断。实际上相当于是输入的所有x相当于一张图，然后生成器生成一个向量表征每个x的判断结果，再一个向量是真实模型预测的结果。判别器就是判断两个向量哪个是真实的哪个是生成器生成的就好。最后两个向量没区别了，就代表generator基本上能代替victim model 了。
    • 看代码（全部），理清代码逻辑
    • 测试代码，发现sythetic部分无问题，最后生成的augmentation data从victim model的检查结果来看也是“好数据”，但用augmentation data训练出来的模型和真实模型的相似度很低，目前来看，要么是模型的问题，要么是dataset的问题，还有待排查。
    • 用augmentation data 作为输入和用synthetic data 和用 real data 作为输入三者最后得到的模型相似度，augmentation data 的远低于另外两个，需要注意。

    • 还有一点，训练GAN的batchsize=500，最后生成的是个2*500的矩阵，是不是太大了点？这样的话如果光看图片的话，很有可能有些点出错也瞧不出来，尝试减少batchsize试试看，batchsize太大我觉得很容易淹没误差信息

      • batchsize=25，adam，epoch=400： 很差，20,100多的时候上过70，但都没到过80, 其他的都在40-60波动，基本上排除是因为epoch不足带来的损失，猜测可能数据上太好了点，都是非零即一的判别，没有教会模型“模糊判别”的重要性，去除掉VAE之后的过滤环节，尝试训练模型。另一方面可能是数据集太小的问题，下次搞个大一些的数据集
