整理一些互联网常见协议

2020-07-04 network network 0 评论 字数统计: 647(字) 阅读时长: 2(分)

汇总一些常见网络协议以及基本解释，以备日后复习查看

[TOC]

数据链路层

PPP (Point-to-Point Protocol)

进行网络通信前后的一些准备工作，比如创建连接、检查数据链路的可靠性、ISP（internet service provider）分配IP、通信结束后断开连接等等

PPP（点对点协议(Point to Point Protocol)）_百度百科

网络层

ARP（Address Resolution Protocol）

通过IP地址查询Mac地址

ARP（地址解析协议）_百度百科

有了 IP 地址，为什么还要用 MAC 地址？ - 知乎

ICMP（Internet Control Message Protocol）

搭建好网络后的一种差错检测机制，ICMP 允许主机或路由器报告差错情况和提供有关异常情况的报告。

ICMP协议详解_honeyJ-CSDN博客_icmp

CIDR（Classless Inter-Domain Routing）

无分类域间路由选择 （特点是消除了传统的 A 类、B 类和 C 类地址以及划分子网的概念，并使用各种长度的“网络前缀”(network-prefix)来代替分类地址中的网络号和子网号）

IGP (Interior Gateway Protocol )

用来在单一自治系统（AS）内决定路由，也就是维护路由表

RIP（Routing Information Protocol）

基于距离矢量路由协议，只能最多进行15跳，用的少

自己的路由表来自于相邻路由器的路由表（抄来的）

RIP协议_百度百科

OSPF（Open Shortest Path First）

基于链路状态路由协议，没有跳数限制。OSPF是典型的链路状态路由协议，路由器之间交换的并不是路由表，而是链路状态，OSPF通过获得网络中所有的链路状态信息，从而计算出到达每个目标精确的网络路径。

OSPF详解 - 知乎

BGP（Border Gateway Protocol）

边界网关协议BGP（Border Gateway Protocol）是一种实现自治系统AS（Autonomous System）之间的路由可达，并选择最佳路由的距离矢量路由协议。
