Configure Vaultwarden

2024-10-03 Arch Linux chatgpt history, crontab, vaultwarden 0 Comments Word Count: 653(words) Read Count: 4(minutes)

Vaultwarden is a lightweight, self-hosted password manager that is compatible with Bitwarden clients. It is designed as an efficient alternative to the official Bitwarden server, using fewer system resources while providing similar features like storing, managing, and sharing passwords securely. Vaultwarden can be run in a Docker container, making it easy to set up on personal servers or low-powered devices. It supports a wide range of Bitwarden clients, including browser extensions, mobile apps, and desktop apps, and can be extended with features like two-factor authentication and encrypted storage.

Configuring

  1. Install it and run it using docker, according to the official tutorial
  2. Create new account, initialize the database and setup the password, etc.
  3. (Optional) Configure firewall using UFW or other tools to allow communication to it.
  4. Install the browser extension, mobile app for connection
  5. Restart the service using command like this to:
    1. disable registration of new users and invitation
    2. Enable auto restart
    3. port remapping
  1. Setup backup for the data folder, vw-data on the example above. Can follow this tutorial in the end

For the usage in LAN, it is not necessary to setup HTTPS.
For private use, using a Wireguard to setup a private network for all your devices can be a better choice.
