Use MITMProxy for capturing Android traffics and analyze

2024-11-16 hacking chatgpt history, debugging 0 Comments Word Count: 1.8k(words) Read Count: 11(minutes)

Capture and analyze https traffics using MITMProxy. It relies on man-in-the-middle and registering certificate to the device.

MITMProxy uses port 8080 for proxy. Before using it, remember to allow port 8080 in ufw sudo ufw allow 8080. And delete the rule after using sudo ufw delete allow 8080

You:
I want to sniff and analyze the traffics of my android applications, for example, I developed one app and I want to track its https communication, how?

ChatGPT:
To sniff and analyze HTTPS traffic from an Android app, you typically have to bypass the app’s SSL/TLS encryption. Here’s a step-by-step guide to do this safely on your local network for testing and development:

1. Use a Proxy Tool

  • Install a proxy tool like Charles Proxy, Fiddler, or Burp Suite on your computer. These tools intercept HTTP and HTTPS traffic and allow you to inspect the payload.

2. Configure Your Android Device to Use the Proxy

  • Connect your Android device and computer to the same Wi-Fi network.
  • Go to Settings > Wi-Fi on your Android device and long-press the connected network to modify it.
  • Under Advanced settings, set up a manual proxy with the IP address of your computer (running the proxy tool) and the port specified in the tool (usually 8888 or similar).

3. Install the Proxy’s SSL Certificate on Your Android Device

  • For HTTPS decryption, you’ll need to install the proxy tool’s SSL certificate on your Android device.
  • Most proxy tools have an option to download the certificate as a .pem or .cer file.
  • Transfer this file to your Android device, go to Settings > Security > Install from storage (or a similar option), and install the certificate as a trusted CA.

4. Set Up the App to Allow HTTPS Inspection

  • Some apps have network security configurations that prevent proxy interception. To bypass this:
    • If you have control over the app (like a self-developed app), add a debug-only network_security_config.xml file in the app’s res/xml directory. This file should disable certificate pinning:
    • Ensure this configuration is only in your debug build and not in production!

5. Capture and Analyze the Traffic

  • With everything set up, open the proxy tool and start capturing traffic.
  • Run your app on the Android device. You should see HTTPS traffic being intercepted in the proxy tool, and you can inspect requests and responses.

Important Notes

  • Avoid intercepting HTTPS for production apps or any third-party apps without permission.
  • Be cautious with sensitive data and only use this setup for apps you control or have permission to test.
  • Android 7.0 (Nougat) and later requires you to explicitly trust user-installed CAs in each app’s network security configuration.

You:
Is there any opensource solution? Or is fiddler secure?

ChatGPT:
Yes, there are open-source tools for sniffing and analyzing HTTPS traffic from Android applications, and these can be secure if used correctly. Here are some open-source options and security considerations for Fiddler.

Open-Source Solutions

  1. MITMProxy

    • MITMProxy is a powerful, open-source, interactive HTTPS proxy designed specifically for testing and debugging network traffic.
    • Features: It provides a console interface and web interface (mitmweb) to inspect traffic. It supports HTTPS decryption, filtering, scripting, and replaying traffic.
    • Setup: Like other proxy tools, it requires you to set up your Android device to use MITMProxy as the network proxy and install its certificate on the Android device.
    • Customizable: It offers Python scripting support, which makes it flexible for custom test cases.

  2. Wireshark

    • Wireshark is an open-source network protocol analyzer that captures network traffic across interfaces and decodes protocols.
    • Limitations: Wireshark doesn’t have native HTTPS decryption like MITMProxy or Fiddler. You’ll need to manually export the SSL session keys from your Android device, which is feasible only if the app supports SSLKEYLOGFILE. This process can be cumbersome, so it’s best suited for network traffic other than HTTPS unless SSL keys are easily exportable.
    • Alternative Use: You can use Wireshark in combination with MITMProxy by running Wireshark on the decrypted traffic generated by MITMProxy.

  3. Frida with Objection

    • Frida is an open-source, dynamic instrumentation toolkit that, with the help of Objection, can bypass SSL pinning in Android apps.
    • Application: Frida can intercept traffic, bypass SSL pinning, and even inspect encrypted network calls within apps. Objection offers pre-made scripts for SSL pinning bypass.
    • Requirements: Rooted Android devices may be required for full functionality, though newer Frida versions allow SSL bypass on non-rooted devices in many cases.

Is Fiddler Secure?

Fiddler, by Telerik (now part of Progress Software), is widely regarded as a secure and reliable proxy tool when obtained directly from its official source and used in a controlled environment. Here are some key security points:

  • Closed Source: Fiddler itself is not open-source, which can be a concern if you require full transparency.
  • Security Risks: Fiddler installs a root certificate authority (CA) certificate to intercept HTTPS traffic, so it’s crucial to only use it on trusted devices and networks.
  • Official Source: Download Fiddler only from its official site (Fiddler) to avoid malicious or altered versions.

If open-source transparency is a priority or you need flexible scripting, MITMProxy is a top choice for Android traffic analysis. However, if you are comfortable with Fiddler’s security and features, it remains a solid choice for HTTPS debugging.

Step 3: Configure the Proxy on Your Android Device

  1. Connect your Android device and computer to the same Wi-Fi network.
  2. Go to Settings > Wi-Fi on your Android device.
  3. Long-press your connected Wi-Fi network and choose Modify network.
  4. Set up a manual proxy:
    • Proxy hostname: Enter your computer’s IP address.
    • Proxy port: Enter 8080 (or the port MITMProxy is running on).
  5. Save these settings. Your Android device will now route its traffic through MITMProxy.

Step 4: Install the MITMProxy Certificate on Android

  1. Open a browser on your Android device and go to http://mitm.it.
  2. Download the Android certificate from this page.
  3. Install the certificate:

    • Open Settings > Security > Install from storage (or similar, depending on your Android version).

    • Select the downloaded certificate and install it as a trusted CA.

      Note: On Android 7.0+ (Nougat) and later, apps may not trust user-installed certificates by default. For apps under your control, you’ll need to update the network_security_config.xml to allow this.

For apps that use SSL pinning (an additional layer of security), MITMProxy might not be able to intercept HTTPS traffic. If this is an app you control:

  1. Add a network_security_config.xml file to allow user-installed certificates:
  2. Make sure this configuration is included in your debug build, not in production.

For third-party apps, you can use Frida and Objection (as mentioned) to bypass SSL pinning.

Step 6: Capture and Analyze Traffic

  1. With everything set up, start your Android app.
  2. View HTTP/HTTPS requests and responses in the MITMProxy web interface (if using mitmweb) or console (if using mitmproxy).
  3. You can inspect headers, payloads, and other details of each request and response.

Step 7: Stop MITMProxy and Remove the Certificate

After testing, you can:

  1. Stop MITMProxy by closing the terminal.
  2. Remove the proxy settings from your Android device’s Wi-Fi settings.
  3. Delete the MITMProxy certificate from your Android device under Settings > Security > Trusted credentials.

This setup should allow you to intercept and analyze the network traffic of your Android applications effectively. Remember to use this only for apps you control or have permission to analyze.
