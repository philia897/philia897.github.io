Arch Linux安装步骤

2024-07-07 Arch Linux archinstall, grub 0 Comments Word Count: 720(words) Read Count: 2(minutes)

只是简单的记录一下安装的步骤。其实还挺简单的。

安装背景

新买的机械革命的翼龙15Pro电脑，我确实是受够windows了，想试试别的操作系统。Ubuntu其实也不错，比较成熟，但咱已经用过挺多次了，这次想试试新系统，遇到问题了也是学习的过程嘛。

配置是32G内存+AMD的CPU+Nvidia 4060显卡 + 2T存储
2T都用的是长江存储，因为是在拼多多上买的特供版，比标准版加了一块1T的P300存储。新加的硬盘没有原装的性能好，不过也够用了倒是。稍微有点浪费，毕竟咱计划是以Arch作为主力系统使用的。不过也问题不大，数据很多还是存在Win系统盘里，反正Arch也能挂载，就还可以。

安装步骤

硬盘准备

先把硬盘进行一个清空，用diskgenius进行格式化，确定硬盘变成未使用状态就行

制作启动盘

用Rufus和Arch的镜像制作启动U盘，随便找个什么U盘，够大就行。

本来是准备用ventoy做启动盘的，这样可以不用浪费我一个U盘，结果用不了，郁闷。

安装

插上U盘之后进入BIOS，从U盘启动放到前边，然后disable掉安全模式，然后启动，就能正常进入Arch的安装界面了，一个经典的CLI

这里就可以按照各种教程来进行正常的安装了，也可以图简单运行archinstall进行prompt安装。

建议选择btrfs文件系统，更新一些并且对分卷和备份很友好
安装的时候建议不安装neofetch了，直接安装fastfetch，因为neofetch停止维护了嘛
安装的时候选择安装库git fastfetch htop
桌面系统可以选择hyprland和KDE两个都装，也可以先只装KDE，hyprland之后手动装也无所谓。
总体上没什么坑，各种库可以晚点再自己装也不迟

Grub

安装完成之后如果没啥问题就可以了，因为这次是装双系统，然后两个系统安装在不同的硬盘里，而Arch是用Grub进行引导的，所以手动修改一下grub配置，然后重新grub-mkconfig一样就行，把windows的盘的启动选项加进grub.cfg里边就行了，这块直接问GPT就可以。总体上思路很清晰

至于Grub主题配置之类的也直接查教程就行。需要注意的是自己写custom配置的时候，/etc/grub.d/文件夹下的custom选项前边的数字决定了启动项的顺序，可以通过这个调整顺序。也可以安装grub-customizer进行配置，但那个是通过proxy进行配置的，原理不太一样，不过都可以。
