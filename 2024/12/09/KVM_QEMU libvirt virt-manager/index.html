How to setup KVM/QEMU on Arch.

I’m an Arch user, and now I want to setup an VM using KVM/Qemu. The VM shall use Arch OS as well and I have prepared the ISO file. Show me how to make it

First of all, remember to disable your firewall! As it will affect the setup of networking in the VM

To set up an Arch Linux virtual machine (VM) using KVM/QEMU, you can follow these steps. Here’s a detailed guide:

Step 1: Ensure KVM/QEMU is Installed

  1. Install the necessary packages:

  2. Enable and start the libvirtd service:

  3. Verify that virtualization is supported:

    Ensure the output shows VT-x (Intel) or AMD-V (AMD).

  4. Add your user to the libvirt group to manage VMs without root:

    Log out and log back in for this to take effect.

Step 2: Prepare the Arch ISO

Ensure you have the Arch ISO file ready. For example, suppose the file is at /home/username/Downloads/archlinux.iso.

Step 3: Create a Virtual Machine

  1. Launch virt-manager:

    This opens the Virtual Machine Manager GUI.

  2. Create a new VM:

    • Click File → New Virtual Machine.
    • Select Local install media (ISO image or CDROM) and click Forward.
    • Choose Use ISO image, browse for your Arch Linux ISO, and click Forward.

  3. Assign resources:

    • Allocate memory (RAM) and CPUs. For example:
      • RAM: 2048 MB (2 GB) or more
      • CPUs: 2 or more
    • Click Forward.

  4. Set up storage:

    • Choose Create a disk image for the virtual machine.
    • Allocate disk space (e.g., 20 GB or more).
    • Click Forward.

  5. Configure advanced options:

    • Set Firmware to UEFI if desired (use edk2-ovmf for Arch).
    • Review the configuration and click Finish to create the VM.

Step 4: Boot the VM

  1. Start the VM:

    • In virt-manager, select the VM and click Start.

  2. Follow the Arch Linux installation process:

    • Boot from the ISO.
    • Partition the virtual disk, configure the system, and install Arch as you would on a physical machine.

Step 5: Network Configuration

KVM uses NAT for network by default, allowing your VM to access the internet. If you need a bridged network for direct access to your local network, configure it in virt-manager under Network Interfaces.

Optional: Configure the VM for Optimal Performance

  1. Install virtio drivers in the Arch VM:

    • During installation, ensure the kernel has virtio modules enabled for better performance (e.g., virtio_blk, virtio_net).

  2. Enable SPICE or QXL for better graphics:

    • In virt-manager, under Display, choose Spice and install the spice-vdagent package in the guest for clipboard and resolution integration.

Let me know if you need guidance on a specific step or if you’d prefer a terminal-only setup!

Ah I have ufw firewall in my host machine, will that affect?

Yes, having ufw (Uncomplicated Firewall) enabled on your host machine can absolutely affect the VM’s ability to communicate with the network, especially when using NAT or bridge mode. By default, ufw blocks some types of traffic that are essential for virtual machine networking, including:

  1. DHCP traffic (UDP ports 67 and 68) for IP address assignment.
  2. DNS traffic (UDP port 53) for name resolution.
  3. NAT or bridge traffic between the VM and the host.

To fix this, you need to adjust the ufw settings to allow the necessary traffic. Here’s how:
