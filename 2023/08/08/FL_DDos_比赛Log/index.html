Federated Learning for DDos 比赛 Log

2023-08-08 Logs Federated Learning, ML, competition, pytorch 0 Comments Word Count: 795(words) Read Count: 3(minutes)

记录比赛实践期间的一些所学和感受，简记学到的知识和理解

2020/10/28

  1. 配置仓库
  2. 数据基础处理

2020/11/8

前情回顾：

  • 加入预处理之后直接起飞，准确率直逼100%，主要矛盾转变为过拟合问题
  • 减小模型层数，使用4层感知机，加入dropout层，效果有所提升
  • 调参，，，略吧，，无聊
  • 聚合算法中加入权重：标签丰富度，根据标签丰富度来进行梯度聚合，效果有所提升
  • 加入学习率周期性衰减，效果有所提升
  • 目前最好测试acc为62.7%

之后的测试在改进的基础上进行

测试：

layer_num = 4
lr=0.001
weight_decay = 0.01
batchsize=1600
n_round_sample=16000
max_round=120
acc_limit = 95

result:

50轮早停，train：55%，test：57%，未过拟合，效果不错的

75轮早停，train：64%，test：60%，误差范围内吧，还算好

100轮早停，train：68%，test：58%，过拟合，有点严重

实验效果不理想呀，怀疑是学习率的问题？我觉得应该不是模型结构的问题，train在上升，但test并没有提高，只能说明这时候产生了过拟合，test的学习率就没有上去过

测试：

layer_num = 4
lr=0.001
weight_decay = 0.1
batchsize=1600
n_round_sample=16000
max_round=100
acc_limit = 95

85轮早停，train：73%，test： 56.68%，过拟合比较严重，原因不明

测试：

layer_num = 4
lr=0.003
weight_decay = 0.1
batchsize=1600
n_round_sample=16000
max_round=100
acc_limit = 95

太惨烈，75轮停止，train：67%，test：47%，过拟合严重，看来调学习率似乎也没办法解决？

测试：

layer_num = 4
lr=0.001
weight_decay = 0.0005
batchsize=1600
n_round_sample=16000
max_round=75
acc_limit = 95

大概是比较默认的值了，不抱希望的看看效果

结果也确实没给我惊喜，train 73%，test 54%，十分意料之中

总结一下，经过这些个几轮测试，我觉得之后重复类似的测试也没啥意义了，问题不在这里，过拟合的问题需要寻求别的出路来解决，不是通过换优化器，加正则化能解决的了的我觉得。可能需要有其他的创新性的设计才行。联邦平均算法无疑是一个很优秀的算法。至少在训练集上表现相当优异

测试：

layer_num = 4
lr=0.01
weight_decay = 0.01
batchsize=16000
n_round_sample=16000
max_round=75
acc_limit = 95
不加标签

47%acc很差，还是要加标签的

2020/11/12

换回曾经的那个不加weightdecay的版本，62%的那个。

参数如下：

layer_num = 4
lr=0.001
batchsize=4800
n_round_sample=4800
max_round=200
acc_limit = 95

然后构造了交叉验证集，诡异的事情发生了：

训练集最后90%，交叉验证集87%，说明并没有过拟合，可能：

  1. 我的交叉验证集还是数据过于集中，看不出过拟合问题
  2. 测试集和训练集的数据分布差异较大，比如测试集里全是那些不容易区分的攻击类型

加入focal loss策略试试看：

layer_num = 4
lr=0.003
batchsize=4800
n_round_sample=4800
max_round=200
acc_limit = 95
focal loss = true

最后差别不大，训练集90%，验证集89%
